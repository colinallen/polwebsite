Completing an Abbreviated Truth Table

After selecting an abbreviated truth table exercise, you are presented with an incomplete abbreviated truth table like this one:

The Power of Logic
Exercise 7.4A Problem 50

The top line of the incomplete abbreviated truth table contains the statement letters contained in the argument, a vertical line, and the sentence or argument for which the truth table is to be constructed. The next line contains a separator -- basically a row of hyphens and a single vertical line. The third line is blank. Begin by filling in a  T beneath the main logical operator of each premise (or beneath any statement letter that occurs as a premise) and an  F beneath the main logical operator of the conclusion (or beneath the conclusion itself, if it is a statement letter):

Now we flesh out the row by assigning truth values appropriately to the component statements in the argument. Specifically, in Layman's method, one proceeds by identifying a truth value assignment for the sentence letters that will make the conclusion false (there will always be one, as the conclusions in the exercises are never tautologies) and then testing whether all of the premises can be made true on that assignment. If so, the argument is invalid. If not, identify any other truth value assignments that make the conclusion false and fill in another row in the same fashion. If, on any of these assignments, the premises can all be made true, the argument is invalid. On the other hand, if, on every truth value assignment that makes the conclusion false, it is not possible to make all of the premises true, the argument is valid.

Given this method, then, a row in an abbreviated truth value is filled out in accordance with the following three rules:

  1. A truth value (i.e., 'T' or 'F') must occur beneath every sentence letter in the argument
  2. A truth value or a slash (i.e., '/') must occur beneath every logical operator in the argument.
  3. A slash can occur only under the main logical operator of a premise.

(Note that, in practice, strict adherence to the first two rules is not required -- in many abbreviated truth tables, some truth values in some rows are irrelevant and can be omitted. The first rule is required here only because it allows for a much simpler design than a looser rule would have permitted.)

Because there are three ways to make (A . C) false, each way must be tried in turn until a consistent assignment for the component statement letters is found (i.e., an "invalidating assignment"), or until all three ways have failed to find such an assignment. All that remains is to to try to calculate the value for ~C and to find a value for B. Since C is false, ~C must be true, so a 'T' will have to go underneath the tilde in ~C. But this means that (B -> ~C) will be true regardless of what truth value we assign to B. However, note that by Rule 1 for completing a row of an abbreviated truth table, a truth value must still be entered for B. We will choose 'T' arbitrarily. But note now that the truth value for entire premise ~(B -> ~C) is F on this assigment of truth values to B and C, which conflicts with our initial assignment of 'T'.
